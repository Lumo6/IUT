Question 2 : Comment B sait qu'il est le destinataire du message envoyé par A ?
Dans un réseau, chaque hôte est identifié par une adresse unique, telle qu'une adresse MAC ou une adresse IP. Quand A envoie un message, celui-ci contient l'adresse de destination (l'adresse de B). B reçoit tous les messages sur le réseau, mais il ne traitera que ceux dont l'adresse de destination correspond à la sienne.

Exercice 2 : Transfert de fichier
Calcul de T1 et T2 :

Taille du fichier : 320 K octets = 320 * 1024 * 8 bits = 2 621 440 bits.
T1 (sans les bits Start, Stop et Contrôle) :
Débit = 9600 bits/sec
T1 = 2 621 440 bits / 9600 bits/sec = 273,08 secondes

T2 (en tenant compte des bits Start, Stop et de Contrôle) :
Pour chaque octet (8 bits), on ajoute 3 bits (Start, Stop, Contrôle), donc 11 bits par octet.
Nombre total de bits avec les bits supplémentaires = 11/8 * 2 621 440 bits = 3 606 980 bits
T2 = 3 606 980 / 9600 = 375,73 secondes

Rapport R = (T2 - T1) / T2 :
R = (375,73 - 273,08) / 375,73 = 0,273 soit environ 27,3 %
Délai total T3 avec valence du signal égale à 4 :

La valence du signal étant 4, cela signifie que chaque signal nécessite 4 unités de temps pour être transféré.
T3 = T2 * 4 = 375,73 * 4 = 1502,92 secondes


Exercice 3 : Connexion numérique via satellite
Calcul du temps de propagation T1 :

Distance entre A et le satellite : 36 000 km
Temps de propagation : T1 = 36 000 km / 300 000 km/s = 0,12 seconde

Le signal doit parcourir 36 000 km pour atteindre B, puis retourner, donc le double : 72 000 km
Temps total pour l'acquittement : T_ack = 72 000 km / 300 000 km/s = 0,24 seconde

Débit = 64 000 bits/sec, donc en 0,24 seconde, A peut émettre :

Bits émis = 64 000 bits/sec * 0,24 sec = 15 360 bits

Calcul de T2, temps d'émission de la chaîne :

Taille de la chaîne = 96 Kbits = 96 000 bits
Temps d'émission T2 = 96 000 bits / 64 000 bits/sec = 1,5 seconde

3. I = 1 0 0 0 0 1 1
Nombre de bits à 1 : 3 (impair)
Bit de parité : 1 (pour garder la parité impaire)

Les lignes avec les bits de parité ajoutés sont donc :
O = 1 0 0 1 1 1 1 1
S = 1 0 1 0 0 1 1 0
I = 1 0 0 0 0 1 1 1
Calcul du LRC (Longitudinal Redundancy Check)
Nous allons maintenant calculer le LRC pour chaque colonne, en vérifiant la parité impaire pour chaque bit verticalement.

Calcul pour chaque colonne :
Colonne 1 : 1 + 1 + 1 = 3 (impair) → LRC = 1
Colonne 2 : 0 + 0 + 0 = 0 (pair) → LRC = 0
Colonne 3 : 0 + 1 + 0 = 1 (impair) → LRC = 1
Colonne 4 : 1 + 0 + 0 = 1 (impair) → LRC = 1
Colonne 5 : 1 + 0 + 0 = 1 (impair) → LRC = 1
Colonne 6 : 1 + 1 + 1 = 3 (impair) → LRC = 1
Colonne 7 : 1 + 1 + 1 = 3 (impair) → LRC = 1
Colonne 8 : 1 + 0 + 1 = 2 (pair) → LRC = 0
Le LRC final est donc : 1 0 1 1 1 1 1 0.
